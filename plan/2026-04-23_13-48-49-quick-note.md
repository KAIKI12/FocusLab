# Plan: 速记便签功能 Demo

## 🎯 任务概述

为 FocusLab 添加速记便签功能，用户可以快速记录灵感和想法（可能逻辑跳跃、只有关键点），然后通过 AI 优化描述，理解真实意图，最终转化为任务或保存为笔记。

**核心价值：** 降低记录门槛，AI 辅助将碎片化想法结构化。

## 📋 实现方案

### 1. 功能定位

**速记便签 vs 快速添加任务：**
- **快速添加任务**（⌘N）：已知要做什么，直接创建结构化任务
- **速记便签**（⌘⇧N）：灵感闪现，先记下来，AI 帮助理解和结构化

**交互流程：**
```
用户按 ⌘⇧N → 弹出速记输入框 → 输入关键点 → 点击"AI 优化" 
→ AI 返回优化后的描述 + 建议操作 → 用户选择：
  1. 创建为任务（跳转到快速添加任务，预填充内容）
  2. 保存为笔记（存入速记历史）
  3. 继续编辑
  4. 放弃
```

### 2. 数据模型（Demo 版本）

**暂不新建数据库表**，使用 localStorage 存储速记历史：

```typescript
interface QuickNote {
  id: string;
  rawText: string;           // 原始速记内容
  optimizedText: string;     // AI 优化后的内容
  suggestedAction: string;   // AI 建议的操作类型
  createdAt: string;
  status: 'draft' | 'converted' | 'archived';
}

// localStorage key: 'fl-quick-notes'
```

### 3. AI 集成方案

**新增 AI 命令：** `ai_optimize_quick_note`

**Prompt 设计：**
```
System: 你是一个专注力管理助手。用户会输入碎片化的想法或关键点，可能逻辑跳跃、不完整。
你的任务是：
1. 理解用户的真实意图
2. 将关键点扩展为完整、清晰的描述（2-3 句话）
3. 判断这是一个"任务"还是"想法/笔记"
4. 如果是任务，建议四象限分类

User: {用户输入的速记内容}

请返回 JSON 格式：
{
  "optimizedText": "优化后的完整描述",
  "type": "task" | "note",
  "quadrant": "important_urgent" | "important_not_urgent" | ... (仅当 type=task 时)
}
```

**前端调用：**

## ⚠️ 风险和注意事项

1. **AI 调用失败**
   - 显示错误提示："AI 服务暂时不可用，请稍后重试"
   - 允许用户直接使用原始内容创建任务

2. **内容过长/过短**
   - 最小长度：5 字符（前端校验）
   - 最大长度：500 字符（超出提示精简）

3. **快捷键冲突**
   - ⌘⇧N 目前未被占用，安全

4. **性能考虑**
   - localStorage 存储上限：最多保留 50 条历史记录
   - 超出后自动删除最旧的记录
