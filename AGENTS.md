# Project AGENTS

## UX-first 开发规则

- 后续所有功能开发、交互设计、重构和体验优化，默认优先考虑用户体验、可理解性、操作效率和低负担使用感。
- 不以“最小开发量”“最少改动文件数”或“最小实现范围”作为首要目标；如果更完整的实现能显著提升用户体验，应优先选择更方便用户的方案。
- 做方案选择时，优先考虑：
  - 是否减少用户来回点击、跳转、记忆负担和重复操作
  - 是否让状态、原因、结果和下一步动作更清楚
  - 是否让高频流程更顺手、更快、更可连续处理
  - 是否降低误操作成本，并提供撤销、恢复、确认或明确反馈
- 可以接受适度增加实现复杂度、组件数量或改动范围，只要这些改动明显改善用户体验，并且风险可控、结构清晰、可验证。
- 仅当更大的实现不会带来明显体验收益，或会引入不成比例的复杂度、风险和维护成本时，才退回较小方案。
