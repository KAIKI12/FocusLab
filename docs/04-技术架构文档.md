## 2. 技术栈明细

### 2.1 前端（WebView 内）

| 类别 | 技术选型 |
|------|---------|
| 框架 | Vue 3 (Composition API + `<script setup>`) |
| 语言 | TypeScript（严格模式） |
| 构建 | Vite 5 |
| 路由 | Vue Router 4 |
| 状态管理 | Pinia |
| UI框架 | 不用第三方UI库,自建组件（保持轻量+风格统一） |
| 样式 | TailwindCSS 3 + CSS Variables（主题切换） |
| 动画 | Vue Transition / CSS Animation / Canvas 2D / Lottie（按需） |
| 图表 | ECharts 5（热力图/趋势图/饼图） |
| 图标 | Lucide Icons（MIT 开源,风格统一） |
| 拖拽 | vuedraggable / @vueuse/core useDraggable |
| 日期 | dayjs |
| Markdown | markdown-it（周报导出预览） |
| 测试 | Vitest + Vue Test Utils |

### 2.2 后端（Tauri / Rust）

| 类别 | 技术选型 |
|------|---------|
| 框架 | Tauri 2.x |
| 语言 | Rust (edition 2021) |
| 数据库 | SQLite 3 via rusqlite / sqlx |
| HTTP 客户端 | reqwest（调用 AI API） |
| JSON | serde + serde_json |
| 异步运行时 | tokio |
| 密钥存储 | keyring-rs（跨平台钥匙串） |
| 日志 | tracing + tracing-subscriber |
| 数据库迁移 | 自建简易迁移系统（SQL 文件 + 版本号） |
| UUID | uuid crate |
| 时间 | chrono |
| 测试 | Rust 内置测试框架 |

### 2.3 开发工具链

| 类别 | 技术选型 |
|------|---------|
| 包管理 | pnpm (前端) + cargo (Rust) |
| 代码规范 | ESLint + Prettier (前端) + clippy (Rust) |

## 3. 多窗口架构

### 3.1 窗口1: 主窗口 (main)

- 类型: 普通窗口,可调节大小
- 默认大小: 1000×700,最小 800×600
- 用途: 完整功能界面（计划/目标/分析/设置）
- 行为: 关闭时隐藏到托盘而非退出应用
- 路由: Vue Router 管理页面切换

### 3.2 窗口2: 悬浮球 (float)

- 类型: 无边框窗口,始终置顶,透明背景
- 大小: 约 200×50 px（动态调整）
- 用途: 显示当前任务 + 计时
- 行为: 可拖拽移动 / 边缘自动吸附 / 单击展开快捷面板
- 特殊属性:
  - `decorations: false`
  - `always_on_top: true`
  - `transparent: true`
  - `skip_taskbar: true`
  - `resizable: false`
- 与主窗口通过 Tauri Event 通信

### 3.3 窗口3: 快捷面板 (quick_panel)

- 类型: 无边框弹出窗口
- 大小: 约 320×400 px
- 用途: 快速操作（切换任务/暂停/查看列表）
- 行为: 从悬浮球位置弹出 / 点击外部自动关闭 / ESC 关闭
- 按需创建和销毁

### 3.4 窗口间通信

- Tauri Event System（Rust → 所有前端窗口）
- 示例事件:
  - `timer:tick` → 每秒广播计时器状态
  - `task:completed` → 任务完成通知
  - `settlement:trigger` → 触发日结算
  - `settings:changed` → 设置变更同步
- 所有窗口共享同一个 Rust 后端状态

### 7.2 数据表详细设计

```sql
-- 长线目标表
CREATE TABLE goals (
    id              TEXT PRIMARY KEY,
    name            TEXT NOT NULL,
    description     TEXT,
    status          TEXT NOT NULL DEFAULT 'active',  -- active | completed | archived
    created_at      DATETIME NOT NULL,
    updated_at      DATETIME NOT NULL,
    completed_at    DATETIME,
    target_date     DATETIME,
    sort_order      INTEGER NOT NULL DEFAULT 0
);

-- 里程碑表
CREATE TABLE milestones (
    id              TEXT PRIMARY KEY,
    goal_id         TEXT NOT NULL REFERENCES goals(id) ON DELETE CASCADE,
    name            TEXT NOT NULL,
    description     TEXT,
    status          TEXT NOT NULL DEFAULT 'pending',  -- pending | in_progress | completed
    sort_order      INTEGER NOT NULL DEFAULT 0,
    created_at      DATETIME NOT NULL,
    updated_at      DATETIME NOT NULL,
    completed_at    DATETIME
);

-- 分类表
CREATE TABLE categories (
    id              TEXT PRIMARY KEY,
    name            TEXT NOT NULL,
    icon            TEXT,
    color           TEXT,
    is_default      BOOLEAN NOT NULL DEFAULT 0,
    sort_order      INTEGER NOT NULL DEFAULT 0
);

-- 任务表
-- [v1.1] plan_date 已移除，由 daily_task_assignments 表替代
-- [v1.1] status 语义调整为「全局生命周期状态」，不再表示某一天的执行状态
-- [v1.1] consecutive_carry_over_days 改名为 carry_over_count，语义更清晰
CREATE TABLE tasks (
    id              TEXT PRIMARY KEY,
    name            TEXT NOT NULL,
    description     TEXT,
    category_id     TEXT REFERENCES categories(id),
    milestone_id    TEXT REFERENCES milestones(id),
    quadrant        TEXT NOT NULL DEFAULT 'important_not_urgent',
                    -- urgent_important | important_not_urgent
                    -- | urgent_not_important | not_urgent_not_important
    urgency_level   TEXT NOT NULL DEFAULT 'ongoing',  -- today_must | this_week | ongoing
    status          TEXT NOT NULL DEFAULT 'pending',
                    -- pending: 待执行（可被拉入每日计划）
                    -- in_progress: 至少开始过一次
                    -- completed: 已完成（全局完成）
                    -- shelved: 已搁置（全局搁置）
    estimated_minutes   INTEGER,
    actual_minutes      INTEGER DEFAULT 0,
    due_date            DATE,
    due_reminder_sent_date DATE,                    -- [v1.2.2] 到期前一天提醒的推送日期(防重复推送)
    is_recurring        BOOLEAN NOT NULL DEFAULT 0,
    recurrence_rule     TEXT,
    source          TEXT NOT NULL DEFAULT 'manual',  -- manual | ai_suggested | carried_over | guided
    shelved_at      DATETIME,
    shelve_reason   TEXT,
    carry_over_count    INTEGER DEFAULT 0,            -- [v1.1] 被顺延的次数
    last_assigned_date  DATE,                         -- [v1.1] 最后一次被分配到哪天的计划
    created_at      DATETIME NOT NULL,
    updated_at      DATETIME NOT NULL,
    completed_at    DATETIME,
    sort_order      INTEGER NOT NULL DEFAULT 0
);

-- 标签表（多对多）
CREATE TABLE tags (
    id              TEXT PRIMARY KEY,
    name            TEXT NOT NULL UNIQUE,
    color           TEXT
);

CREATE TABLE task_tags (
    task_id         TEXT NOT NULL REFERENCES tasks(id) ON DELETE CASCADE,
    tag_id          TEXT NOT NULL REFERENCES tags(id) ON DELETE CASCADE,
    PRIMARY KEY (task_id, tag_id)
);

-- 专注时段表
CREATE TABLE sessions (
    id              TEXT PRIMARY KEY,
    task_id         TEXT NOT NULL REFERENCES tasks(id),
    start_time      DATETIME NOT NULL,
    end_time        DATETIME,
    planned_duration_minutes INTEGER,
    actual_duration_minutes  INTEGER,
    mode            TEXT NOT NULL DEFAULT 'pomodoro',  -- pomodoro | free
    pomodoro_preset TEXT,  -- classic_25 | deep_45 | immersive_90
    status          TEXT NOT NULL DEFAULT 'in_progress',  -- in_progress | completed | abandoned
    is_manual_entry BOOLEAN NOT NULL DEFAULT 0,
    abandon_reason  TEXT,
    created_at      DATETIME NOT NULL
);

-- 中断记录表
CREATE TABLE interruptions (
    id              TEXT PRIMARY KEY,
    session_id      TEXT NOT NULL REFERENCES sessions(id) ON DELETE CASCADE,
    start_time      DATETIME NOT NULL,
    end_time        DATETIME,
```

## 8. 关键技术实现要点

### 8.1 悬浮球置顶 + 透明 + 拖拽 + 吸附

**Tauri 窗口配置**:
```json

  "url": "/float",
  "width": 200,
  "height": 50,
  "decorations": false,
  "alwaysOnTop": true,
  "transparent": true,
  "skipTaskbar": true,
  "resizable": false
}
```

**前端拖拽实现**:
- 监听 mousedown → 开始拖拽
- 监听 mousemove → 调用 Tauri `appWindow.setPosition()`
- 监听 mouseup → 结束拖拽 + 计算吸附位置
- 吸附算法: 距离屏幕边缘 < 20px → 自动贴边,spring 动画缓动
- 半透明: CSS `background-color: rgba(..., 0.7)`,鼠标悬浮时提高到 0.95

### 8.2 全局快捷键

- 使用 `plugin-global-shortcut`
- 注册: `tauri::plugin::global_shortcut::register("CmdOrCtrl+Shift+K", ...)`
- 冲突处理: 注册失败时提示用户更换快捷键
- 用户自定义: 从 settings 表读取快捷键配置

### 8.3 系统托盘

**托盘菜单**: 显示主窗口 / 当前任务状态 / 暂停继续 / 今日进度 / 日结算 / 设置 / 退出

**托盘图标**: 默认应用Logo / 专注中显示计时进度 / 任务完成时短暂闪烁

### 8.4 计时器精度保障

**问题**: 浏览器 setInterval 在标签页不活跃时精度下降

**最终方案**: Rust 后端计时 + 前端 Worker 双保险
- Rust 后端使用 `tokio::time::interval` 每秒广播到前端
- 前端 Web Worker 作为校验
- 两者偏差 > 2秒时以 Rust 为准
- 倒计时结束事件由 Rust 后端触发,保证可靠性

### 8.5 AI 流式响应

- Rust 使用 reqwest 发起 SSE 请求
- 逐 chunk 通过 Tauri Event 推送到前端
- 前端逐字显示（打字机效果）
- 用户可以中途取消生成
- 超时处理: 30秒无响应自动降级为本地话术

### 8.6 数据库迁移

- `migrations/` 目录下按编号存放 SQL 文件
- 数据库中维护 `_migrations` 表记录已执行的迁移
- 应用启动时自动检查并执行未执行的迁移
- 迁移只支持正向（不支持回滚,简化实现）

## 10. 时区与日期分界线设计（v1.1 新增）

### 10.1 逻辑日概念

"逻辑日"（Logical Date）是 FocusLab 中所有按日期统计的基础单位。一个逻辑日 ≠ 自然日（0:00-23:59）,而是从日期分界线到下一个日期分界线。

- 默认分界线: 本地时间凌晨 4:00（研究生常熬夜到凌晨1-2点,这些工作应算"今天"）
- 用户可在设置中自定义（`day_boundary_hour`）
- 示例（分界线=4:00）: 逻辑日"7月10日" = 7/10 04:00:00 ~ 7/11 03:59:59

### 10.2 时间存储规范

| 字段类型 | 存储格式 | 说明 |
|---------|---------|------|
| DATETIME | UTC ISO 8601 `2025-07-10T02:30:00Z` | 所有时间戳 |
| DATE | 逻辑日日期 `2025-07-10` | 计算在写入时完成（Rust后端） |
| TIME | 本地时间 `09:00` | 固定日程等用户设定的时间 |

前端显示时将 UTC 转换为本地时间。

### 10.3 Rust 工具函数

```rust
/// 根据 UTC 时间和用户时区+分界线,计算逻辑日日期
fn to_logical_date(utc_time: DateTime<Utc>, timezone: &Tz, boundary_hour: u32) -> NaiveDate {
    let local_time = utc_time.with_timezone(timezone);
    if local_time.hour() < boundary_hour {
        local_time.date_naive() - Duration::days(1)
    } else {
        local_time.date_naive()
    }
}

/// 获取某个逻辑日的 UTC 时间范围
fn logical_date_range(date: NaiveDate, timezone: &Tz, boundary_hour: u32)
    -> (DateTime<Utc>, DateTime<Utc>)
{
    let start_local = date.and_hms(boundary_hour, 0, 0);
    let end_local = (date + Duration::days(1)).and_hms(boundary_hour, 0, 0);
    (timezone.from_local_datetime(&start_local).unwrap().with_timezone(&Utc),
     timezone.from_local_datetime(&end_local).unwrap().with_timezone(&Utc))
}
```

所有按日期查询的 SQL 都使用 `logical_date_range` 计算的 UTC 范围。

### 10.4 时区变更处理

- 应用每次启动时检测系统时区,如果与 settings 中记录的不同,弹出提示让用户选择更新或保持
- 更新后只影响后续数据,不追溯修改历史数据
- 设置中增加"锁定时区"选项,适合频繁跨时区的用户

## 11. 崩溃恢复与状态持久化（v1.1 新增）

### 11.1 计时器状态持久化

`timer_state` 表只有一行数据,记录当前计时器状态。

**写入时机**: 计时器状态每次变更时立即写入（开始/暂停/继续/休息/完成/放弃）。运行中每30秒更新一次 `updated_at`,用于崩溃后计算丢失时间。写入是原子操作（单行 UPDATE）,性能开销极低。

### 11.2 启动恢复流程

```
应用启动 → 读取 timer_state → status=idle? → 正常启动
                                    ↓ 非idle
                              计算 gap = now - updated_at
                                    │
                    ┌───────────────┼───────────────┐
                    │               │               │
              gap < 2min      2min ≤ gap < 60min   gap ≥ 60min
                    │               │               │
              自动恢复计时    弹出恢复对话框      自动结束session
              提示"已自动    [恢复] [结束] [丢弃]  保存已记录时间
              恢复计时"                            重置为idle
```

### 11.3 日结算补充生成

启动时检查: 如果昨天有任务记录但没有 settlement → 自动生成"兜底结算"（统计数字自动算,AI寄语留空,`trigger_type='auto_backfill'`）。只检查昨天,不追溯更久。

- 数据库不加密（本地风险低）,后期云同步时使用 HTTPS + 端到端加密
- 应用退出时清除内存中的 API Key 缓存

- **AI Prompt 语言**: Prompt 模板支持按语言切换,加一个 language 变量
- **布局适配**: 按钮/标签宽度不写死,使用 flexbox/grid 自适应
