## 1. 设计语言

### 1.1 品牌色彩

**主色**:
| 名称 | 色值 | 用途 |
|------|------|------|
| Primary | #4F8CFF | 宁静蓝,专注、平静 |
| Primary Light | #7AABFF | 悬浮球、轻量元素 |
| Primary Dark | #3366CC | 强调、按钮 |

**辅助色**:
| 名称 | 色值 | 用途 |
|------|------|------|
| Success | #52C41A | 完成绿,任务完成、A级结算 |
| Gold | #FAAD14 | 成就金,S级结算、超额完成 |
| Warning | #FF8C00 | 提醒橙,即将到期、注意 |
| Neutral | #8C8C8C | 中性灰,搁置、未完成（不用红色） |
| Background | #F5F7FA | 浅灰白,主背景 |

**深色模式**:
| 名称 | 色值 | 用途 |
|------|------|------|
| Background | #1A1A2E | 主背景 |
| Surface | #16213E | 卡片背景 |
| Text Primary | #E0E0E0 | 主文字 |
| Text Secondary | #8C8C8C | 次要文字 |

深色模式下所有功能色保持不变,但降低饱和度。

#### 1.1.1 色调主题(v1.2.4)

FocusLab 提供 12 组色调主题,用户可在 `设置 → 外观 → 主题色调` 里全局切换。色调主题与明暗模式(light / dark)**两轴正交**:

| 轴 | `<html>` 属性 | 取值 |
|----|--------------|------|
| 明暗 | `data-theme` | `light`(缺省)· `dark` |
| 色调 | `data-accent-theme` | 12 个主题 id 之一 |

**主题清单**:

| id | 名称 | 氛围 | id | 名称 | 氛围 |
|----|------|------|----|------|------|
| `default` | 🌊 默认蓝 | FocusLab 原版 | `sakura` | 🌸 樱花粉 | 樱花树下 |
| `claude` | ☁️ 奶油陶土 | 咖啡厅暖阳 | `candy` | 🎀 糖果粉紫 | 现代粉紫 |
| `green` | 🌿 护眼绿 | 清晨植物园 | `milktea` | 🧋 奶茶棕粉 | 奶茶店角落 |
| `lavender` | 🪻 薰衣草紫 | 灵感工作室 | `amber` | 🍊 琥珀橙 | 秋日暖阳 |
| `blue-classic` | 🌊 静谧蓝 | 图书馆蓝天 | `teal` | 🦆 水鸭青 | 清澈湖水 |
| `graphite` | 🧊 极简石墨 | 无印良品 | `slate` | 🪨 石板蓝灰 | VS Code 气质 |

**全配色主题化**:品牌、功能色、象限色(q1-q4)、表面、文本、描边、聚焦阴影全部随主题变化。文本变体通过 `color-mix` 派生,`:root` 压暗 25%、`[data-theme="dark"]` 提亮 15%。

**识别色不变**:🌀 自由模式紫罗兰 `#8B5CF6`、🍅 番茄计数点 `#FF7A5C` 作为模式身份色固定不随主题。

**持久化**:`localStorage.fl-accent` + `fl-theme`,跨 tab 通过 `storage` 事件热同步。

完整规格与派生规则见 [`docs/superpowers/specs/2026-04-18-theme-palette-integration-design.md`](superpowers/specs/2026-04-18-theme-palette-integration-design.md)。

## 3. 动效与视觉反馈规范

### 3.1 番茄钟动效

| 状态 | 效果 |
|------|------|
| 倒计时 | 圆环从满到空,顺时针消减 |
| 专注中 | 圆环颜色为 Primary Blue |
| 休息中 | 圆环颜色为 Success Green |
| 最后5分钟 | 圆环轻微脉冲提醒 |
| 完成时 | 圆环填满 + 轻微弹跳动画 |

### 3.2 任务完成动效

- 勾选任务: 复选框弹跳 + 任务文字划线 + 淡出
- 完成音效: 清脆的"叮"声（可关闭）
- 进度条: 平滑增长动画（300ms）

### 3.3 日结算动效

| 等级 | 进度条 | 背景 | 文字 | 音效 |
|------|--------|------|------|------|
| S级 | 彩虹渐变 + 流光动画 | 金色粒子飘散（3秒） | 放大弹入 + 发光 | 成就解锁音效 |
| A级 | 金色 + 发光脉冲 | 星星特效（2秒） | 淡入 + 轻微发光 | 清脆完成音 |
| B级 | 蓝色渐变填充 | 无特效 | 普通淡入 | 柔和提示音 |
| C级 | 灰蓝色填充 | 无特效 | 普通淡入 | 无音效 |

### 3.4 悬浮球动效

| 状态 | 效果 |
|------|------|
| 呼吸效果 | 透明度 70% ↔ 90%,周期4秒,正弦曲线 |
| 🍅 番茄钟进行中 | Primary Blue 边框显示微型 conic 进度环,时间递减 |
| 🌀 自由计时中(v1.2.3) | 紫罗兰 `#8B5CF6` 渐变底 + 虚线外圈,慢速旋转(周期 12s),数字递增 |
| 任务完成 | 短暂的绿色闪烁（0.5秒） |
| 吸附动画 | 靠近边缘时弹性吸附（200ms） |
| 展开面板 | 从悬浮球位置向外扩展（300ms） |

**3.4.1 手势矩阵(v1.2.3)**:

悬浮球手势按"当前是否在计时"分两档,同手势在两种状态下的行为不同,但 🍅 和 🌀 之间**不再分裂**——两种计时态手势一致,只视觉区分。

| 手势 | 🍅 专注中 / 🌀 自由计时中 | 空闲 / 休息时 |
|------|--------------------------|--------------|
| 单击 | 展开 mini 面板 | 展开 mini 面板 |
| **双击** | **⏸ 暂停 / ▶ 继续**(新增) | 打开主窗口 |
| 右键 | 托盘菜单 | 托盘菜单 |
| 拖拽 | 改变位置(松手后吸附边缘) | 同左 |

**设计理由**:
- 原先双击固定为"打开主窗口",对专注中用户是不必要的打断——真正的高频操作是"暂停"。
- 引入"计时中状态双击 = 暂停"后,mini 面板里保留"打开主窗口"按钮做兜底,确保主窗仍可达。
- 自由模式与番茄模式手势一致,避免用户学习两套交互。

### 3.5 页面切换动效

| 场景 | 效果 |
|------|------|
| 侧边栏切换 | 内容区淡入淡出（200ms） |
| 弹窗打开 | 从中心放大 + 背景模糊（300ms） |
| 弹窗关闭 | 缩小 + 淡出（200ms） |
| 引导流程步骤切换 | 左右滑动（300ms） |

### 3.6 通用动画参数

| 类型 | 时长 | 缓动函数 |
|------|------|---------|
| 快速 | 200ms | - |
| 标准 | 300ms | - |
| 强调 | 500ms | - |
| 进入 | - | ease-out |
| 退出 | - | ease-in |

## 4. 隐私与数据安全的用户感知设计

### 4.1 首次 AI 配置隐私提示

用户首次配置 AI API Key 时，必须展示隐私说明弹窗，内容包括：

- "AI 功能需要将你的任务名称和描述发送到 AI 服务商"
- "FocusLab 不会存储或上传任何数据到我们自己的服务器"
- "你可以随时关闭 AI 功能，所有核心功能完全离线可用"

弹窗底部提供两个操作按钮：`[我了解了，继续]` 和 `[暂不使用 AI]`。

### 4.2 AI 调用时的数据最小化原则

| 规则 | 说明 |
|------|------|
| 只发送必要信息 | 任务名称 + 描述 + 统计数据 |
| 不发送完整历史 | 不发送用户的完整历史记录 |
| 不发送其他任务详情 | 每次调用仅包含当前上下文 |
| 不包含个人身份信息 | Prompt 中不包含用户姓名、学校等 PII |
| 可查看发送内容 | 在设置中允许用户查看"发送给 AI 的数据示例" |

### 4.3 本地模型支持路线图预留

Phase 4 中已规划 Ollama 本地模型支持。在当前版本的设置界面中预留"本地模型"入口，标注"即将支持"，让注重隐私的用户知道有这个方向。

### 4.4 隐私设置面板设计

设置 → 隐私 中增加以下选项：

| 选项 | 操作 |
|------|------|
| AI 调用时发送的数据 | `[查看示例]` |
| 清除所有 AI 交互记录 | `[清除]` |
| 导出我的所有数据 | `[导出]`（遵循 GDPR 精神） |

## 5. 异常状态 UI 设计

### 5.1 空状态

各页面在无数据时应展示引导性文案，而非空白页面：

| 场景 | 空状态文案 |
|------|-----------|
| 任务列表为空 | "还没有任务，点击 + 添加你的第一个任务" |
| 长线目标为空 | "设一个长线目标，让每天的努力都有方向" |
| 数据分析无数据 | "使用几天后，这里会出现你的时间分析" |
| 周报无数据 | "本周还没有足够数据生成周报，继续加油" |
| 搁置区为空 | 不显示搁置区（折叠区域自动隐藏） |

### 5.2 加载状态

| 场景 | 加载效果 |
|------|---------|
| AI 生成中 | 骨架屏 + "AI正在思考..." + 打字机光标 |
| 数据分析加载中 | 图表区域骨架屏 |
| 数据库查询慢（>500ms） | 轻量加载指示器 |
| 首次启动数据库初始化 | 全屏加载 + 进度提示 |

### 5.3 错误状态

**AI 调用失败**:

| 错误类型 | 提示文案 | 行为 |
|---------|---------|------|
| 网络错误 | "无法连接到 AI 服务，已使用本地话术" | 降级到本地 |
| API Key 无效 | "API Key 验证失败，请检查设置" | 引导到设置页 |
| 额度用尽 | "API 额度不足，请充值或更换 Key" | 引导到设置页 |

所有 AI 错误都不阻塞核心功能。

**数据库错误**:

| 错误类型 | 提示文案 | 行为 |
|---------|---------|------|
| 写入失败 | "数据保存失败，请重试" | 提供重试按钮 |
| 数据库损坏 | "检测到数据异常，正在尝试修复..." | 自动尝试从最近备份恢复 |
| 磁盘满 | "存储空间不足，请清理磁盘" | 提示用户清理 |

**其他错误**:

| 错误类型 | 提示文案 | 行为 |
|---------|---------|------|
| 快捷键注册失败 | "快捷键 Cmd+Shift+K 被其他应用占用，请更换" | 引导用户更换快捷键 |
| 悬浮球创建失败 | — | 降级为系统托盘模式 |

### 5.4 网络状态

| 场景 | 行为 |
|------|------|
| 离线 | 设置页/AI 功能处显示"当前离线"标识 |
| 离线时 AI 按钮 | 变灰 + tooltip 提示"需要网络连接" |
| 网络恢复 | 不主动弹窗，只恢复按钮可用状态 |

### 5.5 权限问题

| 权限类型 | 触发时机 | 引导文案 |
|---------|---------|---------|
| macOS 辅助功能权限 | 需要全局快捷键增强功能时 | "需要辅助功能权限以支持全局快捷键" |
| 通知权限 | 首次触发通知时 | 请求系统通知权限 |
| 自启动权限 | 设置中开启自启动时 | 引导用户授权 |

### 6.2 关闭的功能

极简模式下以下功能不可用：

- 四象限视图（只有简单列表）
- 引导思考流程
- 长线目标/里程碑
- AI 所有功能
- 热力图/趋势图
- 日结算分级（只显示完成数量，不分 S/A/B/C 级）
- 能量状态选择
- 周报

### 6.3 保留的功能

极简模式下保留以下核心功能：

- 任务列表（纯列表，无分组）
- 番茄钟计时
- 基本日结算（"今天完成了 X 个任务，专注了 X 小时"）
- 悬浮球
- 系统托盘

### 6.4 设置入口

- 设置 → 模式 → `[标准模式]` / `[极简模式]`
- 极简模式下设置页也相应简化
- 随时可以切回标准模式

### 6.5 标准模式的密度控制（v1.2 新增 · **v1.2.1 Superseded**）

> **状态**: 🚫 **Superseded · 本节的具体布局方案不再执行**
>
> 原方案(主屏核心 4 件 + 折叠切片卡 + ≤5 硬约束)在原型对照后被放弃。桌面端横屏布局下,现版 `main-today.html` 的双列布局是合理的,不需要强制压成单列。详见 `docs/06` ADR-010 的 Superseded 说明。
>
> **保留的 soft guideline**:
> - 新增功能时评估是否能并入已有块
> - 实验功能默认关闭(ADR-009)
>
> 以下为原方案内容,仅作历史保留。

**问题背景**(原方案): 标准模式开启所有功能,如果全部同屏呈现,新用户第一眼会感到过载。解决方式**不是砍功能**,而是**分层布局**。详见 `docs/06` ADR-010。

**主屏只显示 4 件核心操作**:

```
┌─────────────────────────────────────────┐
│  [今日任务列表]                          │
│    ☐ 复现baseline代码                    │
│    ☐ 阅读文献2篇                          │
│    ✅ 回复导师邮件                        │
│                                         │
│  [焦点任务卡]   ← 当前关注的一件事      │
│                                         │
│  [▶ 番茄计时]   ← 当前计时状态          │
│                                         │
│  [🌙 结束今天]  ← 进入日结算            │
│                                         │
│  ▾ 今日切片     ← 折叠信号卡,默认收起   │
│                                         │
└─────────────────────────────────────────┘
```

**折叠"今日切片"卡**(默认收起):
- 日期 · 能量状态
- 专注统计 · 四象限切换
- 完成率进度条
- 早/晚心情(若启用 · 实验功能 · 见 `docs/02 §5.6`)

**密度约束**:
- 主屏同时可见信息块 ≤ 5 个
- 四象限分类、长线目标链接、AI 拆解、热力图等**均下沉到二级页或命令面板**
- 折叠卡不参与视觉计数,收起态仅显示 1 行摘要

## 7. 无障碍设计要求

### 7.1 键盘导航

所有可交互元素必须支持键盘操作：

| 按键 | 行为 |
|------|------|
| Tab | 遍历所有可交互元素 |
| Enter / Space | 触发按钮和复选框 |
| 方向键 | 在任务列表中上下移动 |
| ESC | 关闭弹窗/面板 |
| 快捷键 | 覆盖核心操作（参见快捷键设计） |

### 7.2 颜色不作为唯一信息传达手段

- 四象限除了颜色（红黄橙绿）外，必须有文字标签（"紧急重要""不紧急重要"等）
- 日结算评级除了进度条颜色外，有大号文字（S/A/B/C）
- 需验证配色方案对色盲用户的可区分性

### 7.3 字号可调

研究生长时间看屏幕，对字号敏感。提供 3 档字号调节：

| 档位 | 说明 |
|------|------|
| 小 | 紧凑布局，适合大屏 |
| 中（默认） | 标准字号 |
| 大 | 放大字号，适合长时间阅读 |

或跟随系统字号设置。

### 7.4 动画可关闭

部分用户对动画敏感（前庭功能障碍），设置中增加"减少动画"开关：

- 开启后关闭所有粒子特效、呼吸灯、滑动动画
- 保留功能逻辑，只移除视觉运动效果
- 响应系统级 `prefers-reduced-motion` 设置

## 8. 命令面板设计原则（v1.2.2 新增）

**出处**: ADR-010 v1.2.1 "低频功能下沉到命令面板统一入口" · 落地到 `prototype/screens/modals.html` 的 `cmd-palette` modal。

### 8.1 呼出键与定位

| 项 | 值 |
|---|---|
| 呼出键 | `⌘ /`(非 `⌘ K`,避免与浏览器 WebView 默认冲突) |
| 尺寸 | 560px × 最高 420px(含输入框 + body + footer) |
| 层级 | 高于所有其它 modal,触发时自动遮罩其它内容 |
| 关闭 | `ESC` · 点击遮罩 · 回车执行后自动关闭 |
| 状态 | 无持久化 · 每次重新打开是干净状态(但保留"最近 3 条执行历史"在未来迭代可加) |

### 8.2 四类输入模式

| 前缀 | 语义 | 示例 | 优先级 |
|------|------|------|--------|
| (无前缀)  | 混合搜索 · 命中命令名 / 任务 / 目标 / 标签 | `论文` | 通用 |
| `>` | 只过滤命令列表 · 不搜内容 | `> 结束` → 匹配"结束番茄钟"、"结束今天" | 精确 |
| `#` | 只搜任务 · 支持按标签检索 | `#baseline` / `#论文A` | 专注检索 |
| `@` | 只跳转到目标 | `@论文A` | 快速导航 |

**搜索匹配策略**:
- 不区分大小写
- 汉字:支持拼音首字母(如 "jrjh" 匹配"今日计划")
- 模糊匹配:允许字符连续出现即可命中(VS Code 风格)
- 排序:命令(精确匹配) > 置顶任务(今日计划内) > 其它任务 > 目标 > 标签 > 最近使用过的命令

### 8.3 完整命令清单 · 25 条

所有命令在命令面板内按分组排列,保持与 `settings/settings.html` 快捷键分区一致。

**🎯 焦点 / 番茄钟 · 6 条**

| 命令 | 默认快捷键 | 触发条件 |
|------|------|---------|
| 暂停 / 继续专注 | `Space` | 仅番茄/自由计时运行时有效 · 悬浮球**双击**也触发(v1.2.3) |
| 切换任务 | `⌘ T` | 专注中切换到其它任务(记录中断) |
| 结束当前番茄钟 | `⌘ ⇧ X` | 提前结束,进入休息 |
| 切换计时模式 · 🍅 ↔ 🌀 | 未绑定 | 启动前可切;计时中不可切(v1.2.3) |
| 延长休息 5 分钟 | 未绑定 | 休息时可用 |
| 沉浸视图 / 切换悬浮球 | `⌘ ⇧ F` | 主窗口 ↔ 沉浸番茄 / 悬浮球,三态切换 |

**📋 任务 / 计划 · 5 条**

| 命令 | 默认快捷键 | 备注 |
|------|------|------|
| 快速添加任务 | `⌘ N` | 呼出 quick-add modal |
| 启动晨起引导 | `⌘ G` | 进入 morning-guide 5 步流程 |
| 微复盘当前任务 | 未绑定 | 需有当前任务 · 4 种触发条件详见 §docs/02 |
| 设置当前任务截止日期… | 未绑定 | 打开日期选择器 |
| 搁置当前任务 | 未绑定 | 移入搁置区,不再主动提醒 |

**🎨 视图 / 导航 · 5 条**

| 命令 | 默认快捷键 |
|------|------|
| 今日计划 | `⌘ 1` |
| 长线目标 | `⌘ 2` |
| 日历视图 | `⌘ 3` |
| 数据分析 · 周报 | `⌘ 4` |
| 搁置区 | 未绑定 |

**⚙️ 设置 / 模式 · 4 条**

| 命令 | 默认快捷键 |
|------|------|
| 打开设置 | `⌘ ,` |
| 切换极简 / 标准模式 | `⌘ ⇧ M` |
| 切换深色 / 浅色主题 | `⌘ ⇧ T` |
| 查看所有快捷键(即打开本面板) | `⌘ /` |

**💤 日结算 / 收尾 · 2 条**

| 命令 | 默认快捷键 |
|------|------|
| 结束今天 · 进入日结算 | `⌘ ⇧ E` |
| 查看昨日复盘 | 未绑定 |

**🌐 全局 · 1 条**(系统级)

| 命令 | 默认快捷键 | 备注 |
|------|------|------|
| 全局唤起 FocusLab | `⌘ ⇧ L` | 任何应用下可用 · 需操作系统授权 |

**🔍 搜索语义 · 2 条**(不占独立快捷键,靠前缀触发)

- `#` + 关键词 → 搜任务
- `@` + 目标名 → 跳转目标
- (`>` 前缀用于过滤命令,等同于默认搜索)

### 8.4 键位冲突与可自定义

- 所有带 `⌘` 的组合避开浏览器/系统默认(如 `⌘ W` 关窗 · `⌘ R` 刷新 不用作命令)
- 用户可在 `设置 → 快捷键` 分区重新绑定
- 重新绑定冲突时提示"此组合已被 XX 命令占用,是否替换"
- 恢复默认按钮每组末尾一个

### 8.5 设计约束

- **键盘可达**:所有命令必须纯键盘可触发(不依赖鼠标)
- **简洁性**:输入框永远是面板唯一输入源,不要在内部再嵌子输入框
- **不堆功能**:不超过 30 条总命令,超过说明主屏有功能该合并
- **不做向导**:命令是"立刻做"的单步操作,不要包裹成多步流程(那是 modal 的职责)
- **延续 ADR-005**:命令面板中不出现任何"判定"或"批评"语气的命令名
