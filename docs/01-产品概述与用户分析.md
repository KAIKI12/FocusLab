## 1. 产品定位

### 1.1 核心定义

FocusLab 是一款面向在校研究生的桌面端专注力与任务管理应用。它不是一个通用的待办清单工具,而是一个围绕研究生科研节奏设计的「个人工作操作系统」。

**核心理念**: 陪伴式效率管理 —— 不做监工,做镜子;不做法官,做伙伴。

### 1.2 要解决的核心问题

| 问题编号 | 痛点描述 | 现有工具的不足 |
|---------|---------|---------------|
| P1 | 研究生的任务颗粒度模糊,"写论文"太大,不知道今天该做什么 | 通用待办工具不提供任务拆解引导 |
| P2 | 长线任务(如发表论文)缺乏阶段感,容易焦虑或麻木 | 没有里程碑式的长线进度追踪 |
| P3 | 自我管理能力参差不齐,缺乏外部反馈 | 现有工具只记录,不反馈 |
| P4 | 不知道时间花在了哪里,缺乏自我觉察 | 计时工具与任务管理割裂 |
| P5 | 未完成任务带来心理负担,导致逃避甚至卸载工具 | 现有工具用红色标记未完成,加重焦虑 |

### 1.3 产品差异化

**市面上的效率工具**:
- 滴答清单 / Todoist → 通用待办,无科研场景适配
- Forest → 只有专注计时,无任务管理
- Notion → 过于灵活,需要用户自己搭建系统
- 番茄Todo → 番茄钟+待办,但无长线目标追踪
- 以上所有 → 不提供引导思考、AI拆解、情绪感知

**FocusLab 的差异化**:
- ✅ 专为研究生科研节奏设计
- ✅ 引导式每日计划(帮用户想清楚今天做什么)
- ✅ 长线目标 + 里程碑 + 每日小目标的三层映射
- ✅ AI 驱动的任务拆解与个性化建议
- ✅ 温和的未完成任务处理机制(不制造焦虑)
- ✅ 游戏化日结算反馈(但不是游戏)

## 2. 目标用户分析

### 2.1 核心用户画像

**主要用户: 在校研究生(硕士/博士)**

#### 用户画像 A: 硕士研究生小李

- **年龄**: 23岁,研二
- **日常**: 每天在实验室工位工作 8-10 小时
- **痛点**:
  - 导师给了大方向,但不知道每天具体做什么
  - 论文写了两个月,感觉没什么进展
  - 经常被事务性工作打断(报销、填表、开会)
  - 晚上回去觉得"今天好像什么都没做"
- **期望**:
  - 有人帮我理清今天该做什么
  - 能看到自己的长期进展
  - 不要给我太大压力
- **技术水平**: 会用基本软件,不排斥新工具

#### 用户画像 B: 博士研究生小王

- **年龄**: 27岁,博三
- **日常**: 同时推进论文、项目、助教工作
- **痛点**:
  - 多条线并行,经常顾此失彼
  - 长线任务太多,不知道优先级
  - 时间碎片化严重
  - 焦虑感强,效率工具反而加重焦虑
- **期望**:
  - 帮我管理多条并行任务线
  - 告诉我时间花在了哪里
  - 不要让我觉得自己很失败
- **技术水平**: 熟悉各种工具,对产品有较高要求

### 2.2 用户使用场景

#### 场景1: 早晨到工位
1. 打开电脑,FocusLab 自动启动
2. 显示昨日复盘卡片 + 今日引导
3. 用户花 3 分钟设定今日计划
4. 开始第一个番茄钟

#### 场景2: 工作中
1. FocusLab 以迷你悬浮球形式常驻
2. 显示当前任务名 + 番茄钟倒计时
3. 番茄钟结束时,柔和提醒休息
4. 用户切换任务时,一键切换计时

#### 场景3: 被打断
1. 导师突然叫去开会
2. 用户点击"暂停",记录中断原因
3. 回来后点击"继续"或切换到新任务
4. 中断数据被记录,用于分析时间碎片化

#### 场景4: 一天结束
1. 用户点击"结束今天"或到达设定的结算时间
2. 弹出日结算面板,展示完成情况
3. 根据完成度给予分级反馈
4. 用户可以写一句今日感想(可选)

#### 场景5: 周末回顾
1. 用户打开周报功能
2. 查看本周时间分布、完成率趋势
3. AI 生成周度总结和建议
4. 用户调整下周的长线目标计划

### 2.3 用户心理模型

**研究生使用效率工具的心理周期**:

- **第1天**: 兴奋期 → "这个工具好酷,我要好好用"
- **第3天**: 适应期 → "还行,但有些功能用不上"
- **第7天**: 倦怠期 → "每天打开有点烦,但还在用"
- **第14天**: 分水岭 → 要么形成习惯,要么开始遗忘
- **第30天**: 稳定期 → 如果还在用,说明真的有价值

**FocusLab 的留存策略**:
- 第1-3天: 简化流程,不要让用户觉得复杂
- 第3-7天: 开始展示数据洞察,让用户看到价值
- 第7-14天: 引入长线目标追踪,建立沉没成本
- 第14-30天: AI 建议开始基于历史数据,越来越精准
- 第30天+: 用户已经积累了大量数据,迁移成本高

## 3. 产品设计原则

### 3.1 核心原则

#### 原则1: 陪伴,不监工
- 不用红色标记未完成任务
- 不用"失败""落后"等负面词汇
- 用"搁置""调整""重新开始"替代
- 让用户感到被支持,而不是被评判

#### 原则2: 镜子,不法官
- AI 呈现数据事实,不做价值判断
- "你今天在XX上花了3小时" ✅
- "你今天效率太低了" ❌
- 让用户自己得出结论

#### 原则3: 工具,不游戏
- 不做经验值、等级、排行榜
- 成就系统奖励"坚持"而非"完成量"
- 视觉反馈要克制,不要过度花哨
- 核心价值是帮用户管理时间,不是娱乐

#### 原则4: 渐进,不强制
- 所有引导流程都可以跳过
- 功能复杂度随使用时间逐步展开
- 新用户看到简洁界面,老用户解锁高级功能
- 永远给用户"不用这个功能"的选项

#### 原则5: 轻量,不打扰
- 常驻时占用最小屏幕空间
- 提醒用柔和方式(呼吸灯效果,不是弹窗)
- 尊重用户的专注状态
- 用户在深度工作时,应用应该"隐形"

### 3.2 情感设计准则

**语言风格**:
- 温暖但不油腻: "今天辛苦了" ✅ "你是最棒的!!!" ❌
- 诚实但不刺耳: "今天完成了3个任务" ✅ "你只完成了3个" ❌
- 建议但不命令: "可以试试..." ✅ "你应该..." ❌
- 用户可自定义AI语气风格

**视觉风格**:
- 主色调: 柔和的蓝绿色系(专注、平静)
- 成功反馈: 暖色光效(金色、橙色)
- 未完成: 灰色淡化(不是红色警告)
- 整体氛围: 像一个安静的图书馆,不是嘈杂的游戏厅
- 支持深色模式(研究生经常熬夜)
